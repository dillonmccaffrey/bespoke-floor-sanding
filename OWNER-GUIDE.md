## Logging Into the Content Manager

1. Go to **https://bespokefloorsanding.ie/admin**
2. Click **Login with GitHub**
3. If prompted, authorize the app with your GitHub account
4. You'll be taken to the Content Manager dashboard

> **Note**: Your GitHub account must have access to the website repository. Contact your web developer if you can't log in.

---

## Content Manager Overview

Once logged in, you'll see the sidebar with these sections:

| Section | What You Can Edit |
|---------|-------------------|
| **Site Settings** | Business name, phone, email, hero text |
| **Testimonials** | Customer reviews |
| **Gallery** | Before/after project photos |
| **Services** | Service descriptions and features |

---

## How to Add a Testimonial

1. Click **Testimonials** in the sidebar
2. Click the **New Testimonial** button
3. Fill in the details:

   | Field | What to Enter | Example |
   |-------|---------------|---------|
   | **Name** | Customer's first name | "Mary" |
   | **Location** | Town/county | "Dublin" |
   | **Quote** | Their review | "Fantastic work, floors look brand new!" |
   | **Rating** | Stars (1-5) | 5 |
   | **Order** | Display priority (lower = first) | 1 |

4. Click **Publish** in the top right

---

## How to Add a Gallery Item (Before/After Photos)

1. Click **Gallery** in the sidebar
2. Click **New Gallery (Before/After)**
3. Fill in the details:

   | Field | What to Enter |
   |-------|---------------|
   | **Title** | Project name (e.g., "Victorian Hallway") |
   | **Location** | Where the job was done |
   | **Description** | Brief description of the work |
   | **Before Image** | Click to upload the "before" photo |
   | **After Image** | Click to upload the "after" photo |
   | **Order** | Display priority |

4. Click **Publish**

### Photo Tips

- ✅ Use **landscape** (horizontal) photos
- ✅ Keep images under **2MB** each
- ✅ **JPG format** works best
- ✅ Take before/after from the **same angle**
- ✅ Good lighting makes a big difference

---

## How to Edit Services

1. Click **Services** in the sidebar
2. Click on the service you want to edit
3. Update any of these fields:

   | Field | Description |
   |-------|-------------|
   | **Title** | Service name |
   | **Short Description** | Shows on the services list |
   | **Full Description** | Shows on the service detail page |
   | **Icon** | Visual icon (sanding, restoration, finish, commercial) |
   | **Features** | List of bullet points |

4. Click **Publish**

## How to Update Site Settings

### General Settings (Business Info)

1. Click **Site Settings** → **General Settings**
2. Update your:
   - Business name
   - Logo (upload new image)
   - Phone number
   - Email address
   - Tagline

### Homepage Hero

1. Click **Site Settings** → **Homepage Hero**
2. Update:
   - Badge text (e.g., "Trusted Since 2010")
   - Headlines
   - Call-to-action button text
   - Trust points

### Contact Page

1. Click **Site Settings** → **Contact Page**
2. Update:
   - Page title
   - Description text
   - Service areas list

---

## Publishing Your Changes

After making any changes:

1. Click **Publish** (top right corner)
2. Wait 1-2 minutes for changes to appear on the live site
3. Refresh your website to see the updates

### Save as Draft

If you're not ready to publish:
- Click **Save** instead of **Publish**
- Your changes will be saved but not visible on the live site
- Come back later to continue editing

---

## Understanding the Workflow

```
Edit Content → Publish → GitHub (automatic) → Live Website (1-2 min)
```

When you click Publish:
1. Your changes are saved to GitHub (the code storage)
2. The website automatically rebuilds
3. Your changes appear on the live site

## Troubleshooting

### "I can't log in"
- Make sure you're using your GitHub account
- Contact your web developer to check your access

### "My changes aren't showing"
- Wait 2-3 minutes and refresh the page
- Try clearing your browser cache (Ctrl+Shift+R)

### "Image upload failed"
- Check the file is under 2MB
- Make sure it's a JPG, PNG, or WebP file
- Try a different browser

### "I made a mistake"
- Find the content you want to fix
- Edit it and click Publish again
- Previous versions are saved in GitHub history
